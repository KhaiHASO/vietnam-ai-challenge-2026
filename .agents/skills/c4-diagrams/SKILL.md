# Skill: C4 Diagrams

## Purpose
Clarify system architecture (System Context, Container, Component) before detailed implementation.

## Rules
1. Use ASCII diagrams or Mermaid blocks.
2. Call out assumptions and open questions directly in the design artifact.
3. Align components with the existing monorepo structure (`ai_layer`, `backend`, `frontend`).
4. Prevent detailed design drift (do not code or define micro-functions yet).
