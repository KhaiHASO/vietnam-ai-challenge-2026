# Skill: Architectural Decision Records (ADR)

## Purpose
Draft and preserve critical architectural decisions in MADR format outside the temporary change lifecycle.

## Structure
- **Title**: Deciding on [feature/architecture]
- **Status**: Proposed / Accepted / Rejected
- **Context**: What problem are we solving?
- **Options Considered**: List of alternative solutions.
- **Decision**: The selected option and why.
- **Consequences**: The trade-offs and impact.
- **Persist**: Keep ADR files inside `docs/adr/` permanently.
